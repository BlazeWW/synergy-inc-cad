# CAD File Naming Convention

All CAD and exported files must use clear, traceable names.

## Required Format

```text
project-area_document-type_version_student-or-team_date
```

## Examples

```text
synergy-workshop-layout_concept-v01_john-smith_2026-01-12.pdf
synergy-zoning-plan_concept-v02_team-a_2026-01-14.dwg
synergy-main-site-flow_review-v01_team-b_2026-01-15.png
```

## Rules

- Use lowercase letters where possible.
- Use hyphens instead of spaces.
- Include a version number, for example `v01`, `v02`, `v03`.
- Include your name or team name.
- Include the date in `YYYY-MM-DD` format.
- Do not use names such as `final`, `final-final`, `latest`, or `new-new`.
- Do not overwrite another student's work.

## Versioning Guidance

Use a new version when:

- The layout changes meaningfully.
- Feedback has been applied.
- A file is submitted for review.
- A file is being prepared as a final deliverable.

Keep older versions unless instructed otherwise. Version history is part of the learning process.
