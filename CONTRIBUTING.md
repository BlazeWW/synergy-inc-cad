# Contribution Guidelines

## Before Starting Work

1. Read the project brief.
2. Check the latest files and notes.
3. Confirm the task assigned to you.
4. Check whether someone else is already working on the same area.
5. Create or update files in the correct folder.

## When Saving Work

- Use the required naming convention.
- Export review files as PDF, PNG, or JPG where possible.
- Add notes explaining your design decisions.
- Update the revision log when changes are made.
- Use clear commit messages.

## Commit Message Examples

```text
Add office space planning research
Add initial workshop zoning concept
Update main site flow concept v02
Export parking access diagram for review
Add feedback notes for restaurant layout
Revise restricted access zoning concept
```

## What Not To Commit

Do not commit:

- Temporary CAD files
- Autosave files
- Duplicate exports
- Personal files
- Files with unclear names
- Files called `final-final`
- Very large source files unless approved
- Confidential information not meant for the repository

## Review Process

Work may be reviewed through:

- GitHub Issues
- Pull requests
- Informal remote discussions
- Comments on exported PDFs or images
- Revision notes in `06-feedback-and-revisions/`

Feedback is part of the process. Students are expected to apply feedback professionally.
